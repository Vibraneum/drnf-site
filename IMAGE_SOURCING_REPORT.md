### Step 2: Optimize Downloaded Images

**Online Tools**:
- TinyPNG: https://tinypng.com (compress JPG/PNG)
- Squoosh: https://squoosh.app (advanced optimization)

**Target Specifications**:
- File size: < 200KB per image
- Format: JPG for photos, PNG for logos
- Quality: 80-85% compression

**Command Line (if ImageMagick available)**:
```bash
# Resize and compress
convert input.jpg -resize 1920x800 -quality 85 output.jpg

# Batch process
for img in images/loocafe/*.jpg; do
  convert "$img" -quality 85 "$img"
done
```

## PART 10: Troubleshooting & FAQs

### Q: Can I use news article images without permission?

**A**: Yes, for editorial use featuring your own organization. DRNF is the subject of these articles, so using photos for your own website (non-commercial, informational) falls under fair use. Always provide photo credit to the news outlet.

### Q: What if LooCafe images are too large (>5MB)?

**A**: Use online compression tools:
1. Visit https://tinypng.com
2. Upload image (max 5MB)
3. Download compressed version (typically 70-80% smaller)
4. Or use Squoosh: https://squoosh.app for advanced control

### Q: How do I download Instagram images?

**A**:
1. Visit https://www.instagram.com/loocafe/
2. Open desired post
3. Right-click image → "Open image in new tab"
4. Right-click → "Save Image As"
5. Or use Instagram downloader tools (search "Instagram image downloader")

### Q: Stock photos look too generic. What should I do?

**A**:
1. Prioritize authentic partner images (LooCafe, news articles) first
2. Use stock photos ONLY as temporary placeholders
3. Schedule professional photoshoot for real DRNF activities
4. Search stock sites with very specific terms (e.g., "chai india hyderabad" not just "tea")

### Q: Images aren't loading on the website?

**A**: Check:
1. File path is correct (relative to HTML file)
2. File name matches exactly (case-sensitive on Linux servers)
3. Image file format is web-compatible (JPG, PNG, WebP)
4. File size isn't too large (>2MB may timeout)
5. Browser cache (clear and reload)

### Q: How do I check image optimization?

**A**:
1. Open Chrome DevTools (F12)
2. Go to "Network" tab
3. Reload page
4. Filter by "Img"
5. Check file sizes (should be <200KB each)
6. Run Lighthouse audit (Performance score should be >90)
